Driver Monitoring & Emergency Response System

❓ Problem Statement

Thousands of accidents happen due to driver drowsiness or unconsciousness with no fallback on isolated roads. Our system detects real-time signs and responds immediately.

💡 Our Idea

  • Webcam eye tracking for drowsiness/unconsciousness
  • Emergency protocols & autopilot simulation
  • Azure Maps-based alert system
  • Optional emergency contact calling

🧱 System Components

✔ Driver Monitoring — Python + OpenCV + MediaPipe
✔ Emergency Logic — Python
✔ Autopilot Simulation — Python / Pygame
✔ Alerts — Azure Maps API
✔ Web Interface — Azure Maps, JS
✔ Integration — Flask + HTML/JS

🏆 Why Our Solution Works Best

  • Low-cost with basic hardware
  • Real-time video analysis + cloud alerts
  • Modular, scalable, and integration-ready

🔄 How It Works

  1. Webcam detects EAR (eye aspect ratio)
  2. Drowsiness >5s or no face >3s triggers emergency
  3. Simulated autopilot logs actions
  4. Azure Map marks emergency location
  5. Dashboard displays live status & users nearby

🧪 Prototype Preview

  • Flask frontend + Azure Maps integration
  • Simulated emergency & location marker
  • Pygame autopilot animation (optional)
